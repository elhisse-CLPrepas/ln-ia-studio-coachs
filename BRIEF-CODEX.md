# Mission Codex - LN-IA-SC v0.2.0

## Rôle

Tu es Codex dans VS Code. Tu accompagnes le projet **LAB-NUMÉRIQUE-IA - Studio Coachs** après la publication de la landing page webinaire.

La version `v0.1.3` a permis de publier une page no-framework HTML/CSS/JavaScript sur GitHub Pages. La mission suivante consiste à amorcer la version atelier : transformer la page publiée et les documents de cadrage en supports utiles pour animer, partager et améliorer l'atelier LN-IA Studio Coachs.

## État de départ

Le projet dispose déjà de :

- une landing page publiée via `index.html` ;
- une page miroir `webinaire.html` ;
- des visuels dans `assets/img/` ;
- des messages de partage dans `share/` ;
- des documents sources dans `docs/source/` ;
- un dépôt GitHub versionné avec les tags `v0.1.0`, `v0.1.1`, `v0.1.2` et `v0.1.3`.

## Limite stricte

La mission `v0.2.0` reste no-framework.

Ne pas ajouter :

- framework front-end ;
- back-end ;
- base de données ;
- authentification ;
- dépendances lourdes ;
- automatisation complexe.

L'objectif est de produire un atelier clair, partageable et versionné, pas une application complète.

## Objectif v0.2.0

Préparer la base de l'atelier **LN-IA Studio Coachs** :

- clarifier le déroulé pédagogique ;
- transformer la vision en séquences d'atelier ;
- structurer les fiches participant ;
- préparer les consignes animateur ;
- enrichir les données JSON si utile ;
- garder la landing page cohérente avec l'atelier ;
- préparer une release documentée.

## Sources à respecter

S'appuyer sur les documents suivants :

- `guide_nouveaux_comportemant.docx`
- `document_technique_depot_LN_IA_SC.docx`
- `guide_orchestration_LN_IA_SC.docx`
- `guide_chef_atelier.docx`

Thèmes centraux :

- coaching augmenté par IA ;
- New Paradigm IA ;
- changement de mindset ;
- apprentissage de nouveaux comportements ;
- développement moderne assisté par IA ;
- skills IA devenant indispensables ;
- humain augmenté par IA ;
- production versionnée avec VS Code, Codex, Git et GitHub.

## Livrables attendus

1. Créer ou mettre à jour une fiche de déroulé atelier.
2. Créer une checklist animateur inspirée du `guide_chef_atelier.docx`.
3. Structurer les messages de partage dans `share/`.
4. Vérifier que `README.md`, `ROADMAP.md` et `CHANGELOG.md` restent alignés.
5. Préparer les commandes Git pour commit, tag et push de la version `v0.2.0`.

## Critères de validation

- La landing page reste consultable sans serveur.
- Les fichiers HTML, CSS et JS restent simples.
- Les messages de partage sont lisibles et prêts à copier.
- Les documents d'atelier sont compréhensibles par un humain avant intervention de Codex.
- Aucun fichier source utilisateur n'est supprimé sans validation explicite.
- Les changements sont faciles à relire avant commit.

## Workflow recommandé

1. Lire les documents de cadrage.
2. Identifier les tâches de l'atelier.
3. Produire les supports minimum utiles.
4. Vérifier localement.
5. Committer avec un message clair.
6. Taguer `v0.2.0`.
7. Pousser vers GitHub.
